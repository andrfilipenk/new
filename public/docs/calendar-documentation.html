Installation

The calendar library is located in app/Core/Calendar and integrates seamlessly with your application through the Service Provider pattern.

Step 1: Register the Service Provider

Add the CalendarServiceProvider to your bootstrap.php file:

use Core\Calendar\CalendarServiceProvider;

$providers = [
    // ... other providers
    CalendarServiceProvider::class,
];

Step 2: Use the Factory

The library is now ready to use! Access it through the CalendarFactory:

use Core\Calendar\CalendarFactory;

// Create a calendar
$calendar = CalendarFactory::month()->build();
echo $calendar->render();
‚úì That's it! No additional configuration required. The library works out of the box with sensible defaults.

Quick Start

Basic Month Calendar

Create a simple calendar in just two lines:

use Core\Calendar\CalendarFactory;

$calendar = CalendarFactory::month()->build();
echo $calendar->render(); // Outputs SVG

Calendar with Events/Tasks

Add bars to display events or tasks on specific dates:

use Core\Calendar\CalendarFactory;
use Core\Calendar\Models\Bar;
use DateTimeImmutable;

// Create event bars
$events = [
    new Bar(
        id: 'meeting1',
        title: 'Team Meeting',
        startDate: new DateTimeImmutable('2025-10-15 09:00'),
        endDate: new DateTimeImmutable('2025-10-15 10:30'),
        backgroundColor: '#4CAF50'
    ),
    new Bar(
        id: 'sprint1',
        title: 'Development Sprint',
        startDate: new DateTimeImmutable('2025-10-16'),
        endDate: new DateTimeImmutable('2025-10-20'),
        backgroundColor: '#2196F3'
    ),
];

// Build calendar with events
$calendar = CalendarFactory::month()
    ->addBars($events)
    ->enableWeekNumbers()
    ->build();

echo $calendar->toHtml(); // Outputs SVG + JavaScript

Interactive Calendar

Enable date selection and handle user interactions:

$calendar = CalendarFactory::month()
    ->enableSelection()
    ->onDateClick('handleDateSelection')
    ->bindToFormFields(['date' => 'selected_date'])
    ->build();

echo $calendar->toHtml();

Add corresponding JavaScript handler:

function handleDateSelection(date, element) {
    console.log('Selected date:', date);
    alert('You selected: ' + date);
}

View Types

The library supports multiple calendar views for different use cases:

Month View DEFAULT

Display a full month calendar with optional week numbers:

$calendar = CalendarFactory::month()
    ->setDate(new DateTimeImmutable('2025-10-15'))
    ->enableWeekNumbers()
    ->build();

Week View

Show a single week with customizable start day:

$calendar = CalendarFactory::week()
    ->setDate(new DateTimeImmutable('2025-10-15'))
    ->setFirstDayOfWeek(1) // Monday
    ->build();

Multi-Week View

Display multiple consecutive weeks:

$calendar = CalendarFactory::multiWeek(
    new DateTimeImmutable('2025-10-01'),
    4 // Number of weeks
)->build();

Year View

Show an entire year at a glance:

$calendar = CalendarFactory::year(2025)->build();

Day View

Focus on a single day:

$calendar = CalendarFactory::create('single-day')
    ->setView('day')
    ->setDate(new DateTimeImmutable('2025-10-15'))
    ->build();

Custom Period View

Define your own date range:

$calendar = CalendarFactory::custom(
    new DateTimeImmutable('2025-10-01'),
    new DateTimeImmutable('2025-10-31')
)->build();

Data Providers

Data Providers are responsible for fetching and supplying calendar data (events, tasks, bars) from various sources.

Array Data Provider DEFAULT

The simplest provider - pass an array of Bar objects:

use Core\Calendar\DataProviders\DatabaseDataProvider;

$provider = new DatabaseDataProvider(
    db: $container->get('database'),
    tableName: 'events',
    fieldMapping: [
        'id' => 'event_id',
        'title' => 'event_name',
        'start_date' => 'start_time',
        'end_date' => 'end_time',
        'color' => 'display_color',
        'background_color' => 'bg_color',
    ]
);

$calendar = CalendarFactory::month()
    ->withDataProvider($provider)
    ->build();

Interactivity

Make your calendars interactive with built-in event handlers and form bindings.

Date Selection

Allow users to select dates and trigger custom JavaScript handlers:


function handleDateClick(date, element) {
    console.log('Date clicked:', date);
    // date format: 'YYYY-MM-DD'
    
    // Update UI
    document.getElementById('selected-date').textContent = date;
    
    // Make API call
    fetch('/api/events?date=' + date)
        .then(response => response.json())
        .then(data => console.log(data));
}

Range Selection

Enable users to select date ranges (e.g., for booking systems):


function handleRangeSelect(startDate, endDate, element) {
    console.log('Range selected:', startDate, 'to', endDate);
    
    // Update form fields
    document.getElementById('start_date').value = startDate;
    document.getElementById('end_date').value = endDate;
}

Example 4: Multi-View Calendar Controller

                    
Implement view switching in your controller:

                    
                    
public function calendarAction()
{
    $view = $_GET['view'] ?? 'month';
    $date = new DateTimeImmutable($_GET['date'] ?? 'now');
    
    // Fetch events
    $events = $this->eventService->getEvents();
    
    // Create calendar based on view type
    $calendar = match($view) {
        'day' => CalendarFactory::create('calendar')
            ->setView('day')
            ->setDate($date),
        'week' => CalendarFactory::week($date),
        'month' => CalendarFactory::month($date),
        'year' => CalendarFactory::year($date->format('Y')),
        default => CalendarFactory::month($date)
    };
    
    // Add events and build
    $calendar = $calendar
        ->addBars($this->convertToB ars($events))
        ->enableSelection()
        ->build();
    
    $this->view->setVar('calendar', $calendar->toHtml());
    $this->view->setVar('currentView', $view);
}

Best Practices

‚úì Performance Optimization

  • Use DatabaseDataProvider for large datasets to leverage database indexing
  • Implement caching for frequently accessed calendar data
  • Use CallbackDataProvider for lazy loading and on-demand data fetching
  • Limit the number of bars displayed per day to avoid visual clutter

‚ÑπÔ∏è Accessibility

  • Provide meaningful title attributes for bars
  • Use sufficient color contrast ratios in custom themes
  • Include keyboard navigation support in your JavaScript handlers
  • Add ARIA labels for screen reader support

‚ö†Ô∏è Common Pitfalls

  • Don't forget to call build() after configuring the builder
  • Always use DateTimeImmutable instead of DateTime for thread safety
  • Ensure JavaScript handler function names are globally accessible
  • When using range selection, handle edge cases like single-day ranges

Troubleshooting

Calendar Not Rendering

Issue: Calendar output is empty or shows errors.

Solution:

  • Ensure CalendarServiceProvider is registered in bootstrap
  • Check that you're calling build() on the builder
  • Verify date values are valid DateTimeInterface instances
  • Check error logs for exceptions

JavaScript Handlers Not Working

Issue: Click events don't trigger JavaScript functions.

Solution:

  • Ensure handler functions are defined in global scope
  • Check browser console for JavaScript errors
  • Verify you're using toHtml() instead of render() to include scripts
  • Make sure enableClickable() is called if using interactive features

Bars Not Displaying

Issue: Event/task bars are not visible on the calendar.

Solution:

  • Verify bar dates fall within the visible date range
  • Check that bars have valid startDate and endDate values
  • Ensure data provider is returning bars correctly
  • Try using a contrasting backgroundColor for visibility

Theme Not Applied

Issue: Custom or pre-built theme colors are not showing.

Solution:

  • Verify theme class extends Core\Calendar\Styling\Theme
  • Ensure withTheme() is called before build()
  • Check that theme colors are valid hex codes
